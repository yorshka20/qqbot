## 核心原则

宁可不提取，不要提取错。每条记忆都会长期保留，错误或重复的信息比漏提取危害更大。

---

## 归属规则

- 只提取用户**亲口表达**的信息，归属到该消息的 QQ号
- 从用户行为或操作**推断**出的信息，不提取
- 用户提到、转述他人的信息，不归属到发言者
- 多人讨论中无法确定归属的，不提取

---

## 提取标准

**group_facts**（群全局，只记录客观事实）
- topic：持续性的主要话题方向，需在对话中反复出现
- rule：bot 的群级设定、群公告、群规

**user_facts**（用户记忆）
- identity：用户主动陈述的基本属性（职业、所在地、设备等）
- preference：用户明确且清晰表达的长期偏好或厌恶，临时念头和模糊态度不算
- opinion：用户说出口的观点，需有明确评价或论点，随口吐槽和玩笑不算
- relationship：用户提到的重要人物或社会关系
- instruction：用户对 bot 的明确要求

---

## 提取前自查（每条候选信息都要过一遍）

1. 是用户亲口说的，还是从行为推断的？→ 推断的，丢弃
2. 表达清晰明确，还是随口一提、态度模糊？→ 模糊的，丢弃
3. 是长期信息，还是一次性状态？→ 一次性的，丢弃
4. 与现有记忆相比，是否提供了新的细节？→ 完全重复的，丢弃

---

## 输出格式

只输出 JSON，不要任何其他文字：

{
  "group_facts": [
    { "scope": "topic | rule", "content": "简洁描述" }
  ],
  "user_facts": [
    {
      "user_id": "消息前缀中的QQ号",
      "facts": [
        { "scope": "identity | preference | opinion | relationship | instruction", "content": "简洁描述" }
      ]
    }
  ]
}

无新信息时对应数组为空。
