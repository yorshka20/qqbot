## Scope 格式说明

scope 采用分层格式：`core_scope` 或 `core_scope:subtag`
- **core_scope**：核心分类（必须是下方列出的固定类型）
- **subtag**：可选的细分标签（用于更精确的分类）

### Scope 复用规则（极其重要）
1. **必须优先使用已有 scope**：查看上方「已有的 scope 列表」，如果新信息与已有 scope 语义相关，直接归入该 scope
2. **禁止创建语义相近的新 scope**：如已有 `preference:food`，不要创建 `preference:饮食`、`food_preference` 等
3. **只有完全不同的信息类型才可新建 scope**：新建时使用 `core_scope:subtag` 格式，subtag 用英文 snake_case

## 记忆格式说明
每个 scope 用自然语言段落描述，供其他 LLM 直接阅读。
不使用 bullet 列表，同一 scope 内的内容写成连贯的段落或句群。

---

## 处理流程

### Step 1：现有记忆内部去重
阅读每个 scope 的段落，识别重复、冗余或语义相近的表达，聚合改写为简洁、信息完整的自然语言。
- 保留原有全部独立信息，不得遗漏有价值的细节
- 删除仅措辞不同但语义相同的重复表达

### Step 2：scope 归属校验
处理每条信息（现有或新增）前，先确认其 scope 是否符合当前记忆类型：
- 若当前为 user 记忆，遇到 `rule` 内容 → 直接丢弃
- 若当前为 global 记忆，遇到 identity / preference / opinion / relationship / instruction → 不写入（这些属于用户个人记忆）
- instruction 内容需判断是否为**持续性、可复用**的要求；一次性请求 → 丢弃

### Step 3：融合新信息
对每条新 fact，判断如何处理：
- **语义重复**：与现有内容说的是同一件事且无新细节 → 丢弃
- **补充细节**：为现有内容提供新角度或新细节 → 自然融入相关段落
- **直接矛盾**：明确否定了现有某条内容 → 用新信息替换，旧内容不保留
- **全新内容**：与现有所有内容不重叠 → 追加到对应 scope 末尾，或新建 scope

不确定时，融入而不是丢弃。

## 输出格式
按 scope 分组，每个 scope 写一段自然语言，无内容的 scope 省略。
只输出记忆内容本身，不要任何解释或注释。

格式示例：
```
[core_scope]
段落内容

[core_scope:subtag]
段落内容
```

如：
```
[identity]
用户是一名软件工程师，在北京工作。

[preference:food]
喜欢吃川菜，尤其是麻辣口味。不吃香菜。

[preference:music]
喜欢听摇滚乐和电子音乐。
```
