# Bot Schedule

> Bot每次启动时读取此文件，自动同步到任务库。手动添加任务后重启bot生效，或调用 agendaService.syncFromFile() 热更新。

<!-- ──────────────── 任务格式说明 ────────────────
每个任务用 ## 标题 开始。
元数据用列表形式定义，之后的段落文本为"意图"（告诉bot要做什么）。

触发方式（本文件仅支持长期规则）:
  - 触发: `cron 0 8 * * *`          (每天8点)
  - 触发: `onEvent group_member_join` (有人加群时)
一次性任务请通过指令或 DB 创建，不写入本文件。

执行方式（可选，默认为 intent 由 LLM 解读意图）:
  - 执行: `subagent research`        (直接触发 research 子代理)
  - 执行: `subagent group_report`    (直接触发 group_report 子代理)
  子代理预设名称对应 prompts/subagent/{presetKey}/ 目录
  - 执行: `action todo_worker`       (直接执行代码，无 LLM 参与)
  action 处理器在 AgendaInitializer 中注册，跳过 LLM 直接运行业务逻辑

参数（可选，JSON 格式，传递给子代理的任务输入）:
  - 参数: `{"topic": "AI新闻"}`
  参数会合并到子代理的 task input 中，
  同时也作为模板变量注入到 task.txt 模板的 {{key}} 占位符

冷却时间: `30s` / `5min` / `2h` / `86400000`（毫秒）
─────────────────────────────────────────────── -->
