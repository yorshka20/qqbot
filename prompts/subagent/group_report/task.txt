请为当前群生成一份今日群聊每日汇报。

**当前群名称**: {{groupName}}

## 执行步骤

### 第一步：获取群成员信息

调用 get_group_member_list 工具获取群成员列表。记录下每个成员的：
- userId（用于头像渲染）
- nickname 和 card（群名片，优先使用 card 作为显示名）

### 第二步：获取今日聊天记录

调用 fetch_history_by_time 工具获取从今天 00:00 到当前时间的所有聊天记录：
- startTime: "00:00"
- includeBot: false

### 第三步：深度分析聊天内容

仔细阅读所有消息，分析以下内容：

1. **统计数据**：
   - 总消息数、活跃成员数
   - 每小时活跃度：统计每个小时（0-23点）的消息数量
   - 最活跃时段：找出消息最密集的 2 小时时间段（如 "20:00-22:00"）

2. **今日热议话题**（3-5 个）：
   - 提炼主要讨论话题，不要只写"闲聊"
   - 每个话题用 1-2 句话概括讨论的核心观点和争议点
   - 如果有明显的话题转折或冲突，着重提到

3. **活跃成员点评**（3-6 位）：
   - 优先选最活跃和最有趣的成员
   - 结合第一步获取的成员信息，使用准确的 nickname/card
   - 点评要**个性化**：根据这个人今天实际说了什么来写，不要写通用的评价
   - 可以给成员起有趣的"头衔"，如"今日话痨""摸鱼冠军""深水炸弹"等
   - 每人 1-2 句话，幽默但不冒犯

4. **精选发言**（3-5 条）：
   - 挑选最有趣、最有代表性、最有深度、或引发最多讨论的发言
   - content 截取原文最精华的部分（不超过 100 字）
   - **详细评价**：为什么选这条？它为什么有趣/重要？引发了什么反应？（2-3 句话）
   - 评价要具体、有观点，不要泛泛而谈

5. **群聊总评**（3-5 句话）：
   - 总结今天群聊的整体氛围和亮点
   - 提到今天最有意思的一个瞬间
   - 可以用比喻或有趣的类比来形容今天的群聊风格
   - 如果消息量很少，可以调侃一下今天大家都在忙什么

### 第四步：组装报告数据并渲染

将分析结果组装为 JSON，调用 render_group_report 工具。reportData 格式：

```json
{
  "groupName": "使用上方提供的当前群名称",
  "date": "2024-01-15",
  "totalMessages": 128,
  "activeMembers": 15,
  "highlightTimeRange": "20:00-22:00",
  "hourlyActivity": [
    {"hour": 0, "count": 2},
    {"hour": 1, "count": 0},
    ...（0-23小时每小时一条，必须完整24条）
  ],
  "topics": [
    {"title": "话题标题", "summary": "核心观点和讨论概括"},
    ...
  ],
  "memberHighlights": [
    {"userId": "12345", "nickname": "群名片或昵称", "messageCount": 30, "comment": "个性化幽默点评"},
    ...
  ],
  "featuredMessages": [
    {"userId": "12345", "nickname": "群名片或昵称", "content": "原始发言精华部分", "comment": "详细评价（为什么选这条+评价）"},
    ...
  ],
  "totalSummary": "今天的群聊总评..."
}
```

**数据要求**：
- userId 必须使用聊天记录中真实的用户ID（纯数字字符串）
- nickname 优先使用第一步获取的群名片（card），没有则使用 nickname
- hourlyActivity 必须包含 0-23 所有 24 小时，没消息的 count 设为 0
- featuredMessages.content 不超过 100 字
- 所有文本内容不要包含引号和特殊字符（会影响 JSON 解析）

### 第五步：完成

render_group_report 会自动将图片发送到群内。你无需再发送额外消息。
如果报告已成功发送，请简短回复"每日汇报已生成"。
