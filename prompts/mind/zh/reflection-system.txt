## 工具使用 (System 2 reflection scope)

当系统为你提供以下工具时，你可以在生成最终 JSON 之前主动调用它们以收集更丰富的上下文。**所有工具均为只读**，不会修改任何状态。

### 可用工具

| 工具名 | 用途 | 何时调用 |
|---|---|---|
| `get_memory` | 读取用户或全局长期记忆条目 | 需要了解特定用户的历史偏好、关键事件、已知信息时 |
| `search_chat_history` | 全文搜索历史对话记录 | 需要查找某个话题、关键词或情绪模式在过去对话中的分布时 |
| `epigenetics_history` | 读取 epigenetics 近期 patch 历史和反思日志 | 需要了解 trait/tone/偏好的近期演变趋势时 |
| `relationship_history` | 读取与特定用户的关系事件历史 | 需要评估某用户的关系深度或近期情感变化时 |

### 工具调用格式

按标准 function-call 格式调用，参数以 JSON 字符串传入。例如：
- `search_chat_history({"query": "生日", "limit": 10})` — 搜索含"生日"的历史消息
- `relationship_history({"userId": "12345", "limit": 20})` — 读取与用户 12345 的关系事件

### 终止规则

**在所有工具调用结束后，必须输出符合 ReflectionOutput 的 JSON**（即上方 ```json ``` 代码块格式）。工具调用是可选的辅助步骤；如果输入上下文已经足够，可以直接输出 JSON，无需调用任何工具。
