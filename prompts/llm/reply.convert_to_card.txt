## 卡片类型

根据内容选择最合适的一种：

**1. 基础问答 `qa`**
```
{"type":"qa","question":"用户的问题","answer":"回答内容"}
```

**2. 列表/步骤 `list`**
```
{"type":"list","title":"标题","emoji":"📋","items":["第一项","第二项"]}
```

**3. 提示/警告 `info`**
```
{"type":"info","title":"标题","content":"详细内容","level":"info"}
```
`level` 可选值：`info` `warning` `success` `tip`

**4. 对比分析 `comparison`**
```
{"type":"comparison","title":"对比主题","items":[{"label":"维度","left":"选项A","right":"选项B"}]}
```

**5. 知识解释 `knowledge`**
```
{"type":"knowledge","term":"术语","definition":"解释","examples":["例子1","例子2"]}
```
`examples` 可选。

**6. 数据统计 `stats`**
```
{"type":"stats","title":"标题","data":[{"label":"指标","value":"100","highlight":true}]}
```
`highlight` 可选。

## 字符串字段内的 HTML

`answer`、`content`、`definition`、`items` 各项、`left`/`right` 等字符串字段内部请直接写 HTML，不要写 Markdown。

| 标签 | 用途 |
|------|------|
| `<p>` | 段落，多段时每段用 `<p>` 分开 |
| `<strong>` | 关键词、结论、术语加粗 |
| `<em>` | 补充说明轻微强调 |
| `<code>` | 行内代码、命令、变量 |
| `<pre><code>` | 多行代码块 |
| `<h2>` `<h3>` | 内容内小节/子节标题 |
| `<ul><li>` | 无序列表 |
| `<ol><li>` | 有序列表（步骤、排名） |
| `<table>` | 对比、多列数据，可加 `class="content-table"` |
| `<br>` | 行内强制换行，优先用 `<p>` |

禁止：`<script>` `<style>` `<iframe>` 及其他未列出标签。

**排版建议：** 结论先行（先用 `<p>` 概括），再用标题/列表/表格展开细节；关键词用 `<strong>`，命令用 `<code>`。
