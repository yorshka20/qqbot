## 混合排版规则

**段落与卡片如何配合：**
- 原文开头的引入语、背景说明 → `paragraph`
- 紧接着的列表、对比、步骤等结构化内容 → 对应卡片类型
- 段落之间的过渡句、衔接语 → `paragraph`
- 原文结尾的总结语、开放性问题 → `paragraph`

**何时使用 `paragraph`：**
- 连贯的自然语言叙述（分析、推理、解释）
- 过渡句、引导语（如"下面我从 X 角度分析"）
- 总结语、结束语（如"想深入哪个环节可以继续聊"）
- 原文中不适合塞进任何结构化卡片的内容

**何时使用结构化卡片：**
- 原文中已经是列表/枚举形式的内容 → `list`
- 原文中有明确的两方对比 → `comparison`
- 原文中有步骤/流程描述 → `steps`
- 原文中有术语定义 → `knowledge`
- 原文中有数据/统计 → `stats`
- 原文中有核心结论需要醒目展示 → `highlight`
- 问答形式 → `qa`

**注意：**
- 不要强行将叙述性内容塞入不匹配的卡片类型。如果内容本身是连贯分析，用 `paragraph` 就好。
- 卡片内的文字可以基于原文**改写精炼**，但 `paragraph` 中应尽量**保留原文表达**，可做轻微润色。
- 数组中的元素顺序应与原文叙述顺序一致。

## 字符串字段内的 HTML

`content`、`answer`、`definition`、`items` 各项、`left`/`right`、`detail`、`summary` 等字符串字段内部请直接写 HTML，不要写 Markdown。

| 标签 | 用途 |
|------|------|
| `<p>` | 段落，多段时每段用 `<p>` 分开 |
| `<strong>` | 关键词、结论、术语加粗 |
| `<em>` | 补充说明轻微强调 |
| `<code>` | 行内代码、命令、变量 |
| `<pre><code>` | 多行代码块 |
| `<h3>` | 内容内小节标题（不用 `<h2>`） |
| `<ul><li>` | 无序列表 |
| `<ol><li>` | 有序列表（步骤、排名） |
| `<table>` | 对比、多列数据，可加 `class="content-table"` |
| `<br>` | 行内强制换行（优先用 `<p>`） |

禁止：`<script>` `<style>` `<iframe>` 及其他未列出标签。
