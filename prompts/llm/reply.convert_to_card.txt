## 卡片选择与组合规则

**单卡场景**（内容聚焦，用一种卡片即可完整表达）：
- 问答 → `qa`
- 术语解释 → `knowledge`
- 一句核心结论 → `highlight` 或 `quote`
- 数据罗列 → `stats`
- 流程/教程 → `steps`
- 简单列表 → `list`
- 提示/警告 → `info`
- 两方对比 → `comparison`

**多卡场景**（内容有明显分区，或单卡无法完整承载）：
- 先用 `highlight`/`quote` 抛出核心结论，再用 `list`/`steps` 展开细节
- 先用 `qa` 回答问题，再用 `stats`/`comparison` 补充数据
- 卡片数量建议 **1–3 张**，最多不超过 4 张；宁可内容精炼，不要堆砌

## 字符串字段内的 HTML

`answer`、`content`、`definition`、`items` 各项、`left`/`right`、`detail`、`summary` 等字符串字段内部请直接写 HTML，不要写 Markdown。

| 标签 | 用途 |
|------|------|
| `<p>` | 段落，多段时每段用 `<p>` 分开 |
| `<strong>` | 关键词、结论、术语加粗 |
| `<em>` | 补充说明轻微强调 |
| `<code>` | 行内代码、命令、变量 |
| `<pre><code>` | 多行代码块 |
| `<h3>` | 内容内小节标题（不用 `<h2>`） |
| `<ul><li>` | 无序列表 |
| `<ol><li>` | 有序列表（步骤、排名） |
| `<table>` | 对比、多列数据，可加 `class="content-table"` |
| `<br>` | 行内强制换行（优先用 `<p>`） |

禁止：`<script>` `<style>` `<iframe>` 及其他未列出标签。

**排版建议：** 先用 `<p>` 写核心结论，再用标题/列表/表格展开；关键词用 `<strong>`，命令用 `<code>`。
