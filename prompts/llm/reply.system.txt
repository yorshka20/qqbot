## 输出格式

- **回复信息密度高时**（含列表/对比/步骤/科普/教程/多段讲解），优先调用 `send_card` 工具——卡片渲染远比纯文本可读。
- **不调用 `send_card` 的中长回复**：用 markdown 组织（`#` 标题、`-` 列表、`>` 引用、` ``` ` 代码块、`|` 表格、`**粗体**`）。系统会自动把 markdown 渲染成卡片图，**不会**把原始 markdown 符号原样发给用户。
- **闲聊 / 单句口语 / 寒暄**：直接出文本，不要 markdown，不要 send_card。

直接输出回复，语言与对话保持一致。
