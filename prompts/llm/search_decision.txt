## 你的任务

1. **分析信息完整性**：
   - 当前信息是否足够回答用户问题？
   - 是否缺少关键信息？
   - 是否需要最新的、实时的信息？

2. **决定搜索策略**：
   - 如果信息足够：返回 "NO_SEARCH"
   - 如果需要搜索：返回 "SEARCH: <搜索查询>"
   - 如果需要多个搜索：返回 "MULTI_SEARCH:\n查询1: <query1> | <explanation1>\n查询2: <query2> | <explanation2>"

3. **搜索查询要求**：
   - 查询应该具体、明确
   - 针对缺失的关键信息
   - 可以包含多个关键词以提高搜索精度

## 输出格式

**必须严格按照以下格式输出：**

- 如果不需要搜索：`NO_SEARCH`
- 如果需要单个搜索：`SEARCH: <搜索查询>`
- 如果需要多个搜索：
```
MULTI_SEARCH:
查询1: <query1> | <explanation1>
查询2: <query2> | <explanation2>
```

## 示例

**示例1**（信息足够）：
```
NO_SEARCH
```

**示例2**（需要单个搜索）：
```
SEARCH: 明日方舟终末地最新情报 2024
```

**示例3**（需要多个搜索）：
```
MULTI_SEARCH:
查询1: 明日方舟终末地最新情报 | 获取游戏最新动态
查询2: 明日方舟终末地测试时间 | 了解测试相关信息
```

**重要**：
- 必须返回有效的格式
- 不要包含任何其他文字说明
- 搜索查询应该针对缺失的关键信息
