## SN: Avatar Preview Server

### Background

Local preview + LAN streaming enables developers to observe Live2D avatar parameter updates and animation state in real-time from any device on the same network. The bot runs on a server (e.g., a NAS or development machine); the preview page can be opened on a separate laptop or phone connected to the same LAN — no browser extension or USB cable required.

### What was implemented (`src/avatar/preview/**`)

- **`PreviewServer.ts`** — Bun.serve-based HTTP + WebSocket server with `idleTimeout=255`. Serves `GET /` (self-contained HTML preview page) and `GET /health` (JSON with client count). Maintains a `Set` of connected WebSocket clients and broadcasts `frame` and `status` messages to all of them. Caches the latest `PreviewStatus` and sends it immediately to new clients on connect.
- **`types.ts`** — `PreviewConfig`, `PreviewFrame`, `PreviewStatus`, `FrameMessage`, `StatusMessage`, and `PreviewMessage` union type.
- **`index.ts`** — Public re-export barrel.
- **`client/index.html`** — Self-contained preview page. Shows connection state, status metrics (state, fps, activeAnimations, queueLength), and sorted params with progress bars. Derives its WebSocket host from `location.hostname` so it works on LAN.

### WebSocket message shapes

**`frame`** — broadcast on every animation tick with current parameters:
```json
{
  "type": "frame",
  "data": {
    "timestamp": 1713907200000,
    "params": {
      "ParamAngleX": -12.5,
      "ParamAngleY": 8.3,
      "ParamBodyAngleX": -1.2
    }
  }
}
```

**`status`** — broadcast on state changes and periodically:
```json
{
  "type": "status",
  "data": {
    "state": "idle",
    "fps": 45.2,
    "activeAnimations": 2,
    "queueLength": 0
  }
}
```

### LAN configuration

PreviewServer binds to `0.0.0.0:8002` (configurable via `PreviewConfig.host`/`port`). The preview page derives its WebSocket URL from `location.hostname` (same hostname as the page itself), so it automatically connects to the correct LAN IP when opened from another machine.

## SN: Avatar Idle State Machine

### Background

Live2D avatar 需要状态机管理 Bot 的 5 种表现状态（idle / listening / thinking / speaking / reacting），在无 LLM 输出时自动播放呼吸 / 眨眼 / 微动等微动画，状态切换时输出对应的过渡动画 StateNode。

### What was implemented (`src/avatar/state/**`)

- **`types.ts`** — `BotState` 联合类型、`IdleConfig`（`idleIntervalMin` 默认 3000ms / `idleIntervalMax` 默认 8000ms）、`StateNodeOutput` 接口（结构与 compiler 的 StateNode 一致，本模块自定义避免交叉依赖）、`DEFAULT_IDLE_CONFIG`。
- **`idle-animations.ts`** — `IDLE_ANIMATIONS`（blink / head_sway / breathe 三种微动）、`TRANSITION_ANIMATIONS` Record（按状态映射过渡动画，其中 `thinking` 的 duration=0 表示持续到下一次状态切换）。
- **`IdleStateMachine.ts`** — 继承 `EventEmitter`。`transition(newState)` 返回过渡动画列表，进入 idle 时重启随机定时器，离开 idle 时清掉定时器。`start()` / `stop()` 控制定时器。`scheduleNextIdle()` 用 `setTimeout` 链式重排（非 `setInterval`），每次动画结束后重新随机下次间隔。
- **`index.ts`** — barrel 导出 class / 常量 / 类型。

## S3: Avatar Driver Adapter

### Problem

Avatar system needs a pluggable driver layer to send compiled Live2D parameters to a renderer; VTube Studio is the MVP default.

### Solution

New `src/avatar/drivers/` module — abstract `DriverAdapter` (EventEmitter subclass with connect/disconnect/sendFrame/isConnected) + `VTSDriver` implementing VTS WebSocket API: TokenRequest/Authentication handshake, token persisted to `config/avatar/.vts-token`, `InjectParameterDataRequest` with `mode: 'set'`, exponential-backoff reconnect capped at 30s, 30fps drop-frame throttle.

### 涉及文件

- `src/avatar/drivers/types.ts` — `DriverConfig`, `VTSConfig`, `VTSRequest`, `VTSResponse`, `VTSParameterValue`, `DEFAULT_VTS_CONFIG`
- `src/avatar/drivers/DriverAdapter.ts` — abstract `DriverAdapter extends EventEmitter`
- `src/avatar/drivers/VTSDriver.ts` — `VTSDriver extends DriverAdapter`; full VTS WebSocket protocol
- `src/avatar/drivers/index.ts` — barrel export

## SN: Avatar Animation Compiler

### Problem
Avatar 包缺少把 LLM 语义节点 (StateNode) 编译为逐帧 Live2D 参数流的核心
模块；下游 Driver / Preview Server 已存在，急需 Compiler 才能闭环。

### Solution
在全新目录 `src/avatar/compiler/` 下建立纯逻辑模块：
- `types.ts`：StateNode / ActiveAnimation / FrameOutput / ParamTarget /
  EasingType / AnimationPhase / ActionMapEntry / CompilerConfig
- `easing.ts`：5 种 easing 函数 (linear / easeInOutCubic / easeInOutQuad /
  easeOutElastic / easeOutBounce) + `applyEasing` helper
- `action-map.ts`：`ActionMap` 类同步加载 JSON，
  `resolveAction(action, emotion, intensity)` 返回按 intensity 缩放后的
  ParamTarget[]
- `AnimationCompiler.ts`：继承 EventEmitter；60fps 内部 ticker；ASR 包络
  (attack 20% / sustain 50% / release 30%)；多动作加权混合；低通滤波
  (smoothingFactor=0.3)；按 outputFps=30 降采样发射 `'frame'` 事件
- `index.ts`：barrel 导出所有公共 API
- `config/avatar/action-map.json`：8 个预定义动作
  (smile/nod/wave/thinking/sad/angry/surprised/idle_blink)，参数范围符合
  Live2D Cubism 标准

### 涉及文件
- 新建：`src/avatar/compiler/{types,easing,action-map,AnimationCompiler,index}.ts`
- 新建：`config/avatar/action-map.json`

### Solution
1. **tsconfig.base.json** — 新增根级共享编译选项（无 `noEmit`/`allowImportingTsExtensions`）。
2. **git mv** — 用 `git mv` 移动所有受跟踪文件，保留 git 历史；`scripts/local/` 用 `mv`（.gitignored）。

5. **根 tsconfig.json** — 切换为 project references（`files: [], references: [{path: ./packages/bot}, {path: ./packages/webui}]`）。
6. **根 package.json** — `typecheck: "tsc -b"`，`smoke-test`/`test` 用直接路径（不用 `--filter`）保证 cwd=repo root。
7. **scripts/pm2/pm2-bot.sh** — 更新 `exec bun run` 路径。
