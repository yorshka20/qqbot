**根因 / 设计分析**

Task 1 のコアな判断:
- `enqueueAutonomous` と `enqueueTagAnimation` は完全に同じ modulation + jitter パイプラインを通す。これにより persona modulation が両パスに自動適用される。
- `source` フィールドは StateNode に optional で付与 — 既存の state-machine transition nodes との後方互換を保つ。
- `_enqueueModulated` private helper が単一の真実源。两つの公開 API が別々に modulation ロジックを持つ設計ミスを回避。

Task 2 のコアな判断:
- `setRate(multiplier)` は angular frequency を `_rate` 倍するだけ — amplitude/center/endPose に触れない。
- AutoBlinkLayer は既にスケジュールされた `nextBlinkAt` を遡及修正しない — 次の interval から新 rate が適用。これは mind-system が rate を動的変化させる際の「突然のタイミング変化」を避ける設計選択。

**解決策**

統合 fix は不要。Task 1 + Task 2 の diff を確認した結果、APIの衝突・型不整合・テスト重複はなし。
