- `_enqueueModulated` private helper が単一の真実源。两つの公開 API が別々に modulation ロジックを持つ設計ミスを回避。

Task 2 のコアな判断:

- AutoBlinkLayer は既にスケジュールされた `nextBlinkAt` を遡及修正しない — 次の interval から新 rate が適用。これは mind-system が rate を動的変化させる際の「突然のタイミング変化」を避ける設計選択。

## Session: Mind wander + posture wiring, debug-quiet, gaze visibility

**问题描述**

After landing the PersonaPostureLayer feature, the mind subsystem still didn't
drive anything visible beyond `deriveModulation` (fatigue → intensity/speed
scalars already consumed by AvatarService). Two gaps remained:

1. **Fatigue → posture / gaze-contact bias** was computed nowhere — the avatar
   package had a shiny `setPersonaPostureBias` API with no caller.
2. **WanderScheduler** was implemented but not actually started from
   bootstrap, so autonomous "idle glance / micro-step" behavior was dead code.

Bootstrap itself had also grown a long linear sequence of mind↔avatar wire-up
that should have been one call.

User validation feedback after wiring everything in:
- Wander motion *is* visible but stiff.
- Gaze/lookAt changes barely register — want stronger visual signal.
- Perlin micro-perturbations obscure deliberate motion; need a way to pause
  them for clean testing.

**解决策**

1. **`startMindSubsystem()` single entry point** in
   [packages/bot/src/mind/MindLifecycle.ts](packages/bot/src/mind/MindLifecycle.ts):
   wraps `attachAvatar`, `startPostureDriver` (1 Hz push), and WanderScheduler
   start. Bootstrap collapses from ~55 lines to one call.
2. **`derivePersonaPostureBias(phenotype)`** in
   [packages/bot/src/mind/ode.ts](packages/bot/src/mind/ode.ts): baseline
   `postureLean=0.08`, `gazeContactPreference=0.6`; fatigue adds lean × 0.25
   and subtracts gaze × 0.3. Stays inside documented ranges for any fatigue.
3. **`WanderScheduler.gateBlockReason()`**: returns debug strings
   (`'avatar-not-active'`, `'pose-is-thinking'`, `'cooldown-800ms-remaining'`)
   so `LOG_LEVEL=debug` surfaces *why* wander isn't firing.
4. **`compiler.debugQuiet` config flag** in `CompilerConfig`: when true,
   `AnimationCompiler` skips PerlinNoiseLayer registration and calls
   `EyeGazeLayer.setQuietMode(true)`, which freezes the OU drift + saccade
   loop so only explicit `setGazeTarget()` overrides move the eyes. The
   override path always wins regardless of the flag. Logged as
   `continuous stack → LayerManager: [debugQuiet]` so the effect is visible
   in boot logs.
5. **`glance` intent upgrade** in
   [packages/bot/src/mind/wander/intents.ts](packages/bot/src/mind/wander/intents.ts):
   for `left`/`right` targets, emit a head-yaw `turn()` step *before* the
   gaze and a mirrored `turn()` after the hold, so the whole face rotates
   instead of just the eyeballs. `up` and `camera` targets skip the head
   turn (no mapping from pitch / no-rotation). Hold duration extended from
   800–2200 ms to 2500–4000 ms so the pose is readable.
