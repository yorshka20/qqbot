## 群聊汇报功能 Bug 修复

**任务**: `/report` 命令触发后 bot 无反应，汇报未发送。

**问题分析** (通过 logs/2026-03-24/2026-03-24-01-47-49.log):
1. **`get_group_member_list` 工具失败** (line 629, 637): "Unexpected API response format"
   - 原因: Milky API 返回 `{ members: [...] }` 对象，而代码用 `Array.isArray(result)` 检查直接数组
2. **使用了错误的 LLM provider**: SubAgent 使用了全局配置的 Groq/qwen3-32b，而非可靠的 deepseek
3. **系统提示词未加载**: SubAgentExecutor 将 `analysis` 类型映射到 `subagent.research.system`，而非 `subagent.group_report.system`
4. **LLM token 限制过低**: SubAgentExecutor 硬编码 `maxTokens: 1500`，不足以生成完整的报告 JSON 数据

**修复方案**:
1. `GetGroupMemberListToolExecutor.ts`: 兼容 `{ members: [...] }` 和直接数组两种响应格式
2. `SubAgentRolePresets.ts`: group_report preset 添加 `providerName: 'deepseek'`, `maxTokens: 8000`, `maxToolRounds: 8`, `systemTemplate`
3. `SubAgentExecutor.ts`: 支持从 session config 读取 `maxTokens`、`maxToolRounds`、`systemTemplate` 覆盖
4. `types.ts`: SubAgentConfig 新增 `maxTokens`、`maxToolRounds`、`systemTemplate` 可选字段

**涉及文件**:
- `src/tools/executors/GetGroupMemberListToolExecutor.ts` - 修复 API 响应格式处理
- `src/plugins/plugins/MessageTriggerPlugin/SubAgentRolePresets.ts` - 增加 provider/token 配置
- `src/agent/SubAgentExecutor.ts` - 支持 config 覆盖硬编码参数
- `src/agent/types.ts` - 扩展 SubAgentConfig 接口
