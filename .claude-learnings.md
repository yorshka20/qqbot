## 架构概览

- **卡片渲染管线**: CardRenderer 使用 BrowserService (Puppeteer) 将 HTML 模板通过 `page.setContent()` 渲染为 JPEG 截图。插件也可复用同一模式（如 GroupReportPlugin）。
- **SubAgent 系统**: 通过 `SubAgentRolePresets` 注册预设，prompts 目录中创建 `{presetKey}/keywords.txt` + `task.txt` + `system.txt`，工具通过 `allowedTools` 白名单控制可用性。
- **Schedule 系统**: `data/agenda/schedule.md` 为人类可编辑的调度配置，启动时自动同步到 DB。支持 cron / once / onEvent 三种触发方式。
- **工具注册**: `@Tool()` 装饰器 + `src/tools/executors/index.ts` 导出即自动注册。visibility 控制可见域（reply/subagent/internal）。
- **命令注册**: `@Command()` 装饰器 + `src/command/handlers/index.ts` 导出即自动注册。
- **外部API服务模式**: 独立服务放 `src/services/<name>/`，包含 `types.ts` + `Service.ts` + `executors/` + `index.ts`。服务在 `ConversationInitializer.createCoreServices()` 中实例化并 `container.registerInstance()` 注册，命令和工具通过 `@inject('ServiceName')` 获取。

## 代码模式

- **工具直接发送消息**: 工具可注入 `MessageAPI` (DITokens.MESSAGE_API)，直接通过 `sendGroupMessage()` 发送图片等 MessageSegment，而非依赖 LLM 回复。这适用于生成图片等不能通过文本回复传递的场景。
- **工具调用协议API**: 工具可注入 `APIClient` (DITokens.API_CLIENT)，通过 `apiClient.call()` 调用 milky/onebot11 协议API（如 `get_group_member_list`）获取群信息。
- **QQ 头像 URL**: `https://q1.qlogo.cn/g?b=qq&nk={userId}&s=140`
- **Puppeteer 渲染**: viewport 1000x2000+, deviceScaleFactor=2, JPEG quality=85, `waitUntil: 'domcontentloaded'`（不要用 `networkidle0`，外部图片加载应通过 `page.evaluate` + `Promise.allSettled` 单独处理）。
- **Bilibili WBI 签名**: 搜索和推荐接口需要 WBI 签名（MD5），密钥从 `/x/web-interface/nav` 获取并缓存 12 小时。所有请求需带 `Referer: https://www.bilibili.com` + Chrome UA。

## 常见陷阱

- PromptManager 方法是 `getTemplate()` 不是 `get()`
- `data/` 目录被 `.gitignore` 忽略，schedule.md 等配置文件不会提交
- Milky API `get_group_member_list` 返回 `{ members: [...] }` 而非直接数组，使用 `apiClient.call()` 时需注意 `MilkyAPIResponseHandler` 只提取外层 `data` 字段
- SubAgentExecutor 硬编码 `maxTokens: 1500`、`maxToolRounds: 5`，对于需要生成大 JSON 的任务（如群报告）远远不够。通过 SubAgentConfig 的 `maxTokens`/`maxToolRounds` 可覆盖
- SubAgent 的 system prompt 按 type 映射（`analysis` → `subagent.research.system`），自定义 preset 需通过 `systemTemplate` 字段指定正确的 prompt key
- **Puppeteer `networkidle0` 陷阱**: `page.setContent()` 使用 `networkidle0` 会等待所有网络请求完成，外部 CDN 图片（如 QQ 头像 `q1.qlogo.cn`）慢/不可达会直接导致 30s 超时。应使用 `domcontentloaded` 并单独处理图片加载
