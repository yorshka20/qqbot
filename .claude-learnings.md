## 架构概览

- **卡片渲染管线**: CardRenderer 使用 BrowserService (Puppeteer) 将 HTML 模板通过 `page.setContent()` 渲染为 JPEG 截图。自定义渲染器（如 GroupReportRenderer）也可复用同一模式。
- **SubAgent 系统**: 通过 `SubAgentRolePresets` 注册预设，prompts 目录中创建 `{presetKey}/keywords.txt` + `task.txt` + `system.txt`，工具通过 `allowedTools` 白名单控制可用性。
- **Schedule 系统**: `data/agenda/schedule.md` 为人类可编辑的调度配置，启动时自动同步到 DB。支持 cron / once / onEvent 三种触发方式。
- **工具注册**: `@Tool()` 装饰器 + `src/tools/executors/index.ts` 导出即自动注册。visibility 控制可见域（reply/subagent/internal）。
- **命令注册**: `@Command()` 装饰器 + `src/command/handlers/index.ts` 导出即自动注册。

## 代码模式

- **工具直接发送消息**: 工具可注入 `MessageAPI` (DITokens.MESSAGE_API)，直接通过 `sendGroupMessage()` 发送图片等 MessageSegment，而非依赖 LLM 回复。这适用于生成图片等不能通过文本回复传递的场景。
- **工具调用协议API**: 工具可注入 `APIClient` (DITokens.API_CLIENT)，通过 `apiClient.call()` 调用 milky/onebot11 协议API（如 `get_group_member_list`）获取群信息。
- **QQ 头像 URL**: `https://q1.qlogo.cn/g?b=qq&nk={userId}&s=140`
- **Puppeteer 渲染**: viewport 1000x2000+, deviceScaleFactor=2, JPEG quality=85, `waitUntil: 'networkidle0'` 用于等待图片加载。
